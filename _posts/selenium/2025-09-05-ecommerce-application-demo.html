Designing an End-to-End Selenium Test for E-commerce Automation

Welcome back to the Selenium automation series! To start designing a robust framework, we first need a solid, end-to-end test case.

Using a Sample E-commerce Application

We'll use the Rahul Shetty Academy demo e-commerce site as our example app. Important: Use the URL ending with /client. Avoid going directly to authentication or login URLs to prevent errors.

Test Flow Overview

  • Create a dummy user account on the site with any random email and password you can remember.
  • Login with your created credentials.
  • Select a product (e.g., Zara Coat 3) from the product listing page visible after login.
  • Validate that the "product added to cart" message appears as confirmation.
  • Navigate to cart and confirm the selected products are displayed.
  • Fill in the checkout form with your personal details and select your country by typing to search.
  • Place the order and ensure you land on the confirmation page with a thank-you message.
  • Grab the order ID(s) from the confirmation to use later for verification.
  • Verify orders in the order history tab to ensure all orders you placed are listed.
  • View order details for accuracy of information such as country and other form data.
  • Delete orders and confirm success messages and removal from the list.
  • Return to shopping to repeat or place additional orders.

This test scenario covers a full user journey from login, product selection, checkout, to order history verification.
