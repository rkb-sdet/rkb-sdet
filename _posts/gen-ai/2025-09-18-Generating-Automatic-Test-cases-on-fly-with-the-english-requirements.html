Evolving Test Case Generation: Plain English to Automation with testRigor

TestRigor takes QA productivity to the next level by letting users generate comprehensive manual and automated test cases for each feature update—simply by describing requirements in plain English. This demo illustrates how new feature requirements, like coupon code validations, are instantly transformed into logically grouped tests and, with one click, into automated runs.

Manual and AI-Driven Test Case Generation

  • Start by updating your TestSuite with a new feature description, such as: check coupon field validations.
  • Let testRigor AI suggest the full range of cases: valid coupon, expired/invalid cases, empty field, and error messages.
  • Review, edit, or build these tests into automation scripts—TestSuiteRunner executes and reports results with minimal human intervention.

How testRigor Converts Requirements to Steps

  • Initially, test developers use TestRigorConventions—specific, natural-language action patterns like enter "FREE50" into "Coupon Code" or click "Apply".
  • With evolving AI capabilities, you can describe steps in your own terms. The AIGenerator translates free-form English into the standardized testRigor format, then to automation code.
  • This layered conversion means even non-technical team members or business analysts can write high-level requirements and see them executed as working tests.

Advantages of the testRigor Approach

  • Human-centric: Reduces learning curve for non-coders via conventions, or enables pure English case design with AI translation.
  • Codeless QA: ScriptEngine generates robust test logic—no selector or code-writing required.
  • Continuous learning: AI adapts to varied inputs, increasingly understanding unique ways people describe steps and validations.
  • Stability & self-healing: LocatorResolver automatically adapts to UI changes for ultra-stable regression testing.

AI-powered vs. General LLMs (e.g. ChatGPT, Gemini)

Unlike general purpose LLMs, which require specific HTML or locator details, testRigor’s purpose-built QA stack actually interacts with live web pages—navigating, scraping, and acting on elements using a deep understanding of test flows and app structure. This enables true ContextualAutomation with no manual locators.

General LLMs like ChatGPT and Gemini are best for generic code generation and learning, but testRigor is specialized for business-driven UI automation at scale.
