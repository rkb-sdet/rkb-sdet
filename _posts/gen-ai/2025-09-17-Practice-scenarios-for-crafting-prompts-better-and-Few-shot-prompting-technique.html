Few-Shot Prompting: Using Examples to Guide AI

In the previous sections, we explored zero-shot prompting and how refining prompts with context, constraints, and clarity improves responses. Now, let’s look at another practical technique: Few-Shot Prompting. Instead of just giving AI detailed instructions, you supply examples of the expected output format. This helps the model understand exactly how you want your results structured.

1. Why Few-Shot Prompting Matters

Many testers often give vague prompts like "Write test cases for login page." The problem is that AI might generate random or overly broad test cases. Instead, by providing examples of expected structure, you guide the AI to deliver focused and formatted outputs aligned to your needs.

In a real sprint environment, testers don’t usually write all test cases for an entire system at once. Instead, they focus on specific stories, like valid login, invalid login, or forgot password. Few-shot prompting helps in such cases by showing AI not only what to generate but also how to present it.

2. Adding Constraints for Better Test Case Output

  • Specify the role: "You are a QA engineer working on an e-commerce site."
  • Define scope: focus on valid login, invalid login, and forgot password.
  • Ask for positive and negative test cases.
  • Include environment constraints: e.g., "Browsers: Chrome & Firefox."
  • Request a specific format (e.g., a table with defined columns).
  • Add priority levels for each test case (High, Medium, Low).
Example Refined Prompt:
"You are a QA engineer testing an e-commerce site. Write test cases only for login functionality (valid login, invalid login, forgot password). Include positive and negative scenarios. Cover Firefox and Chrome browsers. Provide results in a table format with columns: Test Case ID, Scenario, Steps, Expected Result, Browser, Priority."

3. What is Few-Shot Prompting?

Few-shot prompting means providing a couple of sample outputs in advance so the AI knows how to structure subsequent answers. This saves time and avoids repeatedly clarifying your needs.

Here’s how it works:

  1. You ask a question: "Write test case for valid login."
  2. You provide an example test case in the format you expect (a table or steps).
  3. The AI uses both your question and your example to generate test cases for other scenarios (invalid login, forgot password, etc.).

4. Example of Few-Shot Prompting for Login Test Cases

Sample input for few-shot prompting could look like this:

Example Prompt with Format:
"You are a QA engineer. Here is the format I want you to follow:"

Test Case ID Scenario Steps Expected Result Browser Priority
TC001 Valid Login 1. Launch site
2. Navigate to Login
3. Enter valid credentials
4. Click Login 		User is successfully logged in and redirected to Dashboard Chrome, Firefox High

"Now generate test cases for Invalid Login and Forgot Password using the same format."

5. Benefits of Few-Shot Prompting

  • Consistency: AI follows the exact structured format you provide.
  • Time-Saving: Avoids converting free-text into tables later.
  • Customization: Ensures prompts align with your team’s reporting standards.
  • Selective Scope: Limits output only to what you actually want.

6. Key Takeaways

  • Few-shot prompting uses examples to guide output formatting.
  • Always include role (context), scope (constraints), and structure (clarity).
  • This technique is highly effective when writing structured outputs, like test cases.
  • Combining technical skills with smart prompting gives you an edge in QA and automation.
