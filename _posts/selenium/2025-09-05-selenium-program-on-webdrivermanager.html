Creating a Standalone End-to-End Selenium Test

In this lecture, we begin by writing a standalone Selenium test before converting it into a structured framework later.

Project Setup

  • Delete default generated files like App.java and AppTest.java from your Maven project.
  • Write test scripts inside the src/test/java package.
  • Use src/main/java for reusable utilities and page objects later.

Creating the Test Class

Create a new Java class named StandAloneTest under the test package. This class will contain the end-to-end automation script.

Using WebDriverManager for ChromeDriver

  • Import the WebDriverManager dependency in pom.xml (not included by default).
  • This tool downloads and manages the ChromeDriver compatible with your browser version automatically.
  • Instantiate ChromeDriver with:
WebDriverManager.chromedriver().setup();
WebDriver driver = new ChromeDriver();

Test Workflow Steps

  1. Invoke browser and open the URL using driver.get("https://rahulshettyacademy.com/client").
  2. Set implicit wait globally to handle sync issues: 
    driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
  4. Locate email and password input fields by id using driver.findElement(By.id("userEmail")) and driver.findElement(By.id("userPassword")).
  5. Send your created user credentials using sendKeys().
  6. Locate and click the login button by its id using click().

Tips & Notes

  • This standalone test illustrates core Selenium functionality without framework features.
  • You can solve product selection logic on your own or wait for the next lecture with a solution.
  • Focus on learning locator strategies, waits, and interactions before introducing framework patterns.
